# LinterGo


## Что делает
1. Находит вызовы log/slog
2. Проверяются их аргументы в соответствии с правилами
3. Нарушения репортятся


## Как делает
1. Драйвер загружает пакеты
2. сортирует их топологически
3. для каждого пакета 
   - создает структуру Pass
   - выполняет Analyzer.Run
   - собирает ошибки в сообщения
4. выводит полученные сообщения

## Строение
1. Analyzer - ядро
2. Driver - точка входа
3. Правила
4. Тесты

## Правила
1. Лог-сообщения должны начинаться со строчной буквы\
   обычная проверка на строчную букву
2. Лог-сообщения должны быть только на английском языке\
   проверка rune на латиницу с пом unicode
3. Лог-сообщения не должны содержать спецсимволы или эмодзи\
   По идее, сообщение должно состоять только из латиницы, так что проверка на isLetter
4. Лог-сообщения не должен содержать потенциально чувствительные данные\
   Тут полагаю, что лучше сделать черный список слов, вынести его в конфиг 

На всякий случай, после первой проверки на строчную букву буду работать с строкой, приведенной к LowerCase для того, чтобы не упустить чувствительные слова

Каждое правило является независимой проверкой, т.е. к одному логу может получиться несколько поправок
